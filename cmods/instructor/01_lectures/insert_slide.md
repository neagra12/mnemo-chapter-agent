## Slide 9: Code Example: A `Joke` Model
- **Key Point:** The ORM "maps" our Python **Model** (a class) directly to a database **Table** (a SQL schema).
- This Python code you write...
- ...is automatically translated into this SQL DDL (Data Definition Language) by SQLAlchemy.

| **Python (Our `models.py`)** | **SQL (What the DB runs)** |
| :--- | :--- |
| ```python # In models.py from app import db class Joke(db.Model): id = db.Column( db.Integer, primary_key=True ) text = db.Column( db.String(280), nullable=False ) rating = db.Column( db.Integer, default=0 ) user_id = db.Column( db.Integer, db.ForeignKey('user.id') ) ``` | ```sql -- Generated by SQLAlchemy CREATE TABLE joke ( id INTEGER NOT NULL, text VARCHAR(280) NOT NULL, rating INTEGER DEFAULT 0, user_id INTEGER, PRIMARY KEY (id), FOREIGN KEY(user_id) REFERENCES user (id) ); ``` |

- **Speaker Note:** This is the magic of the ORM. We write object-oriented Python; it manages the SQL. Point out how `db.Column` maps to the SQL column types and `db.ForeignKey` creates the `FOREIGN KEY` constraint.
